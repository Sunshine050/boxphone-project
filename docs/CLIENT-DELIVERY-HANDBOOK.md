## 1. ภาพรวม: ระบบนี้ทำอะไร

| ส่วน | ใช้ทำอะไร |
|------|------------|
| **เว็บแอดมิน** | จัดการเครื่อง จัดการผู้ใช้ ดูภาพรวม ดู log |
| **เว็บผู้ใช้** | ลูกค้าเข้าสู่ระบบ เลือก/ใช้งานเครื่องที่ได้รับสิทธิ์ |
| **เซิร์ฟเวอร์หลังบ้าน (Backend)** | เก็บข้อมูล ตรวจสอบสิทธิ์ เชื่อมกับมือถือและเสี่ยวเหว๋ย |
| **มือถือ Android** | เครื่องให้เช่า/ให้บริการ — ต่อสาย USB กับคอมที่รันหรือเชื่อมผ่านซอฟต์แวร์เสี่ยวเหว๋ย |

**สิ่งสำคัญ:** ภาพหน้าจอบนแดชบอร์ดส่วนใหญ่ดึงผ่าน **ADB** จากเครื่องที่ **ต่อ USB กับคอมเครื่องเดียวกับที่รัน Backend** (หรือตามสถาปัตยกรรมที่ตกลงกัน) — รายละเอียดอยู่ในคู่มือ Screenshot

## 2. สิ่งที่ต้องเตรียมก่อนติดตั้ง

### ฮาร์ดแวร์ / อุปกรณ์

- [ ] คอมพิวเตอร์หรือเซิร์ฟเวอร์สำหรับรัน **Backend + (ถ้ามี) MongoDB**
- [ ] มือถือ Android แต่ละเครื่อง — เปิด **โหมดนักพัฒนา + USB Debugging**
- [ ] สาย USB คุณภาพดี (ข้อมูลได้) ต่อระหว่างมือถือกับคอม
- [ ] อินเทอร์เน็ตคงที่ สำหรับเว็บและ (ถ้าใช้) คลาวด์

### ซอฟต์แวร์ (ตามที่ใช้จริงในโปรเจกต์)

- [ ] **เสี่ยวเหว๋ย (Xiaowei Desktop)** บนเครื่องที่ต่อมือถือ — ถ้าต้องการ sync รายชื่อเครื่องผ่านเสี่ยวเหว๋ย / ควบคุมตามที่ซอฟต์แวร์รองรับ  
- [ ] **ADB (Android Platform Tools)** บนเครื่องที่รัน Backend — **จำเป็น** สำหรับดึงภาพหน้าจอแบบที่ระบบใช้อยู่  
- [ ] **Node.js** (เวอร์ชันตามที่ทีมพัฒนากำหนด) บนเครื่องรันเว็บและแบ็กเอนด์  
- [ ] **MongoDB** บนเครื่องเดียวกันหรือบริการเช่น MongoDB Atlas  

### บัญชีและข้อมูล

- [ ] โดเมน / HTTPS (แนะนำสำหรับ production)
- [ ] รหัสแอดมินเริ่มต้น (ตั้งในไฟล์สภาพแวดล้อมของ Backend — ไม่แชร์สาธารณะ)

## 3. ลำดับการติดตั้ง (แนะนำ)

ทำตามลำดับนี้จะสับสนน้อยที่สุด

| ลำดับ | งาน | หมายเหตุ |
|------|-----|----------|
| 1 | ติดตั้ง **MongoDB** ให้พร้อม แล้วได้ **สตริงเชื่อมต่อ (URI)** | เก็บเป็นความลับ |
| 2 | ตั้งค่า **Backend** — คัดลอกจาก `backend/.env.example` เป็น `.env` แล้วแก้ค่าให้ครบ | ดูรายละเอียดใน `docs/DEPLOY.md` |
| 3 | ติดตั้ง **ADB** และทดสอบ `adb devices` | ดู `docs/SCREENSHOT-SETUP.md` |
| 4 | ติดตั้งและเปิด **เสี่ยวเหว๋ย** (ถ้าใช้) — ตั้งค่า URL ใน `.env` ของ Backend | ดู `XIAOWEI_SETUP.md` |
| 5 | รัน Backend แล้วทดสอบว่าเข้าถึง API ได้ | เช่น health / login แอดมิน |
| 6 | Build และรัน **เว็บแอดมิน** และ **เว็บผู้ใช้** — ตั้ง `NEXT_PUBLIC_API_BASE_URL` ให้ชี้ไปที่ Backend | ดู `docs/DEPLOY.md` |
| 7 | สร้างผู้ใช้ทดสอบจากแอดมิน แล้วลองล็อกอินฝั่งผู้ใช้ | |

## 4. เสี่ยวเหว๋ย (Xiaowei) ต้องติดตั้งไหม

**ขึ้นกับว่าลูกค้าใช้งานแบบไหน**

- **ถ้าต้องการให้ระบบดึงรายการเครื่อง / ทำงานร่วมกับซอฟต์แวร์เสี่ยวเหว๋ยตามที่ออกแบบไว้** → **ควรติดตั้งเสี่ยวเหว๋ยบนเครื่องที่ต่อมือถือ** และตั้งค่า `XIAOWEI_WS_URL` / `XIAOWEI_API_URL` ใน Backend ให้ตรงกับที่แอปเสี่ยวเหว๋ยเปิดรับ  
- **ภาพหน้าจอบนแดชบอร์ด (แบบที่คู่มือปัจจุบันอธิบาย)** ใช้ **ADB** เป็นหลัก — เสี่ยวเหว๋ยไม่ใช่ตัวส่งภาพโดยตรงในส่วนนี้  

**สรุปง่ายๆ:**  
- ติดตั้งเสี่ยวเหว๋ย = ช่วยเรื่อง **ควบคุม/ซิงก์ตามความสามารถของเสี่ยวเหว๋ย**  
- ติดตั้ง ADB = **จำเป็น** สำหรับ **ภาพหน้าจอ** ตามคู่มือ Screenshot  

เอกสารเสี่ยวเหว๋ยเพิ่มเติมในโปรเจกต์:

- `XIAOWEI_SETUP.md` — ตั้งค่าทั่วไป  
- `XIAOWEI_TROUBLESHOOTING.md` — แก้ปัญหา  
- `XIAOWEI_ACTIVATION_REQUIRED.md` — เรื่อง VIP / เปิดใช้งาน  
- `XIAOWEI_VERIFICATION_CODE_TROUBLESHOOTING.md` — รหัสยืนยัน  
- เอกสารทางการ: <https://www.xiaowei.xin/help/70/349>

## 6. คู่มือใช้งานฝั่งผู้ใช้ (ลูกค้าปลายทาง)

เหมาะสำหรับพิมพ์หรือส่งเป็น PDF ให้ลูกค้าร้านเกม/ผู้เล่น

1. **เปิดเว็บผู้ใช้** ตามลิงก์ที่ร้านให้ (เช่น `https://...`)  
2. **เข้าสู่ระบบ** ด้วย username / password ที่แอดมินสร้างให้  
3. **หน้าแดชบอร์ด** — ดูเครื่องที่ได้รับสิทธิ์ เวลาคงเหลือ และหน้าจอ (ถ้าระบบแสดง)  
4. **เมื่อหมดเวลา** — ระบบอาจคืนสถานะเครื่องและรีเซ็ตหน้าจอ (กลับหน้า Home) ตามการตั้งค่าในระบบ  
5. **ถ้าเข้าไม่ได้** — ตรวจว่าใส่รหัสถูก, ลองรีเฟรช, ติดต่อแอดมินร้าน — ดู [FAQ](#9-แก้ปัญหาที่พบบ่อย-faq)

---

## 7. คู่มือใช้งานฝั่งแอดมิน (เจ้าของร้าน)

1. **เปิดเว็บแอดมิน** ตาม URL ที่ตั้งไว้  
2. **ล็อกอิน** ด้วยบัญชีแอดมิน (รหัสเริ่มต้นตั้งจากไฟล์ environment ของเซิร์ฟเวอร์ — ควรเปลี่ยนหลังติดตั้ง)  
3. **ซิงก์เครื่องจากเสี่ยวเหว๋ย** (ถ้าใช้งานฟีเจอร์นี้) — เมื่อเสี่ยวเหว๋ยเปิดและเชื่อมต่อ  
4. **จัดการผู้ใช้** — สร้าง user, มอบหมายเครื่อง, เติมเวลา  
5. **ดูภาพรวม / Log** — ตรวจสอบเหตุการณ์และสถานะเครื่อง  

*รายละเอียดปุ่มแต่ละหน้าอาจเปลี่ยนตามเวอร์ชัน UI — ให้ทีมพัฒนาแนบภาพหน้าจอเพิ่มถ้าต้องการคู่มือแบบคลิกตามเลขขั้น*

## 8. การ Deploy เว็บ (สรุปให้ทีม IT)

- คู่มือเทคนิคเต็ม: **`docs/DEPLOY.md`**  
- สิ่งที่ต้องคู่กันเสมอ:  
  - **HTTPS**  
  - **`CORS_ORIGINS`** บน Backend = รายการ URL ของเว็บ **ผู้ใช้ + แอดมิน**  
  - **`NEXT_PUBLIC_API_BASE_URL`** บนเว็บ = URL ของ **API**  
  - **`TRUST_PROXY=true`** เมื่ออยู่หลัง Nginx

## 9. แก้ปัญหาที่พบบ่อย (FAQ)

| อาการ | สิ่งที่ลองก่อน |
|--------|----------------|
| เปิดเว็บแล้วขึ้นว่าต่อเซิร์ฟเวอร์ไม่ได้ | ตรวจว่า Backend เปิดอยู่, URL ใน `.env.local` ถูกต้อง, โดเมนใช้ `https` ตรงกับที่ deploy |
| ล็อกอินแล้วเด้งออก / ไม่มี session | ใช้ HTTPS ใน production, ตรวจ `CORS_ORIGINS`, คุกกี้ถูกบล็อกจากเบราว์เซอร์หรือไม่ |
| ไม่เห็นภาพหน้าจอเครื่อง | ทำตาม **`docs/SCREENSHOT-SETUP.md`** — `adb devices`, Serial ตรงกับในระบบ |
| เสี่ยวเหว๋ยไม่ต่อ / ไม่เห็นเครื่อง | อ่าน **`XIAOWEI_TROUBLESHOOTING.md`**, **`XIAOWEI_CHECK_PORT.md`** |
| ขึ้นเรื่อง VIP / เปิดใช้งานเสี่ยวเหว๋ย | **`XIAOWEI_ACTIVATION_REQUIRED.md`**, **`XIAOWEI_BYPASS_VIP.md`** |
| รหัสยืนยันเสี่ยวเหว๋ยไม่มา | **`XIAOWEI_VERIFICATION_CODE_TROUBLESHOOTING.md`** |
| แอดมิน seed ผิดพลาดตอนเปิดเซิร์ฟเวอร์ | ตรวจ `ADMIN_USERNAME` / `ADMIN_PASSWORD` ใน `.env` — เวอร์ชันล่าสุดของระบบจะ **อัปเดตรหัสแอดมินตาม .env** เมื่อมี user ชื่อนั้นอยู่แล้ว |
