# คู่มือเตรียมประชุมลูกค้า — Deploy, โดเมน, งบประมาณ

> ใช้เป็น **สคริปต์/โครงประชุม** ตอนส่งมอบ — ปรับตัวเลขราคาให้ตรงที่คุณเสนอจริง (ราคาคลาวด์เปลี่ยนทุกเดือน)

---

## 1. สิ่งที่ควรเสนอในการประชุม (ลำดับแนะนำ)

| ลำดับ | หัวข้อ | จุดประสงค์ |
|------|--------|------------|
| 1 | **ภาพรวมระบบ** | เว็บแอดมิน + เว็บผู้ใช้ + เซิร์ฟเวอร์ + ฐานข้อมูล + มือถือที่ร้าน (ADB/เสี่ยวเหว๋ย) |
| 2 | **อะไรอยู่ “บนคลาวด์” vs “ที่ร้าน”** | ลดความเข้าใจผิดว่า “จ่ายคลาวด์แล้วมือถือไปอยู่บนนั้น” |
| 3 | **ตัวเลือกแพ็กเกจ 3 ระดับ** | ถูก / กลาง / สูง — ให้ลูกค้าเลือกตามงบ |
| 4 | **โดเมน + HTTPS** | ใครจด, ใครชำระต่อปี, ทำไมต้องมี |
| 5 | **ค่าใช้จ่ายรายเดือน vs ครั้งเดียว** | แยกให้ชัด |
| 6 | **ขั้นตอนหลังประชุม** | ลูกค้าต้องเตรียมอะไร (บัญชีโดเมน, อีเมลติดต่อ, วิธีชำระเงินเซิร์ฟเวอร์) |

---

## 2. สคริปต์สั้นๆ — “บนคลาวด์ vs ที่ร้าน”

**พูดประมาณนี้ได้:**

> “ส่วนที่ลูกค้าเห็นเป็นเว็บ และระบบจัดการหลังบ้าน จะอยู่บน **เซิร์ฟเวอร์ (โฮสต์)** ที่เราตกลงกัน — มีค่าใช้จ่ายรายเดือนหรือรายปี  
> ส่วน **มือถือที่ร้าน** ยังอยู่ที่ร้าน ต่อสาย USB กับคอม — ใช้ **ADB / โปรแกรมเสี่ยวเหว๋ย** ตามที่ติดตั้งไว้ ไม่ได้ย้ายมือถือไปอยู่บนคลาวด์”

---

## 3. แพ็กเกจเสนอ 3 ระดับ (ปรับราคาเองในช่องว่าง)

### แพ็กเกจ A — **ประหยัด (Budget)**

| รายการ | แนวทาง |
|--------|--------|
| เซิร์ฟเวอร์ | VPS ตัวเล็ก–กลาง (เช่น Hetzner / ผู้ให้บริการราคาดี) **เครื่องเดียว** รัน API + เว็บ + MongoDB บนเครื่องเดียวกัน |
| โดเมน | ลูกค้าจดเอง หรือให้ทีมคุณจดแล้วโอนให้ |
| SSL | Let’s Encrypt ฟรี (ผูกกับ Nginx) |
| Backup | สำรองแบบพื้นฐาน (สคริปต์ / snapshot VPS) — **ต้องบอกลูกค้าชัดว่าไม่ใช่แบบ enterprise** |

**ข้อดี:** งบต่ำ, เริ่มใช้เร็ว  
**ข้อเสีย:** ถ้าเครื่องพังหรือโดนโจมตี ความเสี่ยงสูงกว่า; scale ขยายผู้ใช้เยอะๆ ต้องย้ายแผนทีหลัง

---

### แพ็กเกจ B — **มาตรฐาน (แนะนำส่วนใหญ่)**

| รายการ | แนวทาง |
|--------|--------|
| เซิร์ฟเวอร์ | VPS กลาง–ใหญ่ขึ้นตามจำนวนเครื่อง/ผู้ใช้ |
| MongoDB | **MongoDB Atlas** แพ็กเกจเริ่มต้น (แยกจากเครื่อง API) — backup/อัปเดตดีขึ้น |
| โดเมน + HTTPS | เหมือน A |
| Backup | Atlas + snapshot VPS หรือ backup ไฟล์ตามที่ตกลง |

**ข้อดี:** สมดุลราคา–ความมั่นคง; ฐานข้อมูลไม่หายง่ายเมื่อเครื่อง API พัง  
**ข้อเสีย:** แพงกว่า A ต่อเดือนเล็กน้อย–ปานกลาง

---

### แพ็กเกจ C — **สูงขึ้น (ถ้าลูกค้าเน้นความต่อเนื่อง)**

| รายการ | แนวทาง |
|--------|--------|
| แยกชั้น | API บน VPS / เว็บบน Vercel หรือ Cloudflare Pages + DNS/Proxy ที่ Cloudflare |
| DB | Atlas แพ็กเกจที่มี backup ชัดเจน |
| Monitoring | แจ้งเตือนเมื่อเว็บล่ม (บริการเสริมหรือ uptime robot) |

**ข้อดี:** ยืดหยุ่น, แยกความเสี่ยง, เหมาะแบรนด์ที่ต้องการความน่าเชื่อถือสูง  
**ข้อเสีย:** ค่าใช้จ่ายและความซับซ้อนในการดูแลสูงขึ้น

---

## 4. โดเมน — ต้องเสนออะไรบ้าง

| หัวข้อ | สิ่งที่บอกลูกค้า |
|--------|------------------|
| **ทำไมต้องมี** | URL สวย, จำง่าย, ใช้ **HTTPS** กับ cookie/login ได้ถูกต้อง |
| **ใครเป็นเจ้าของ** | แนะนำให้ **ลูกค้าเป็นเจ้าของโดเมน** (จดชื่อลูกค้า) — คุณช่วยตั้งค่าได้ |
| **ค่าใช้จ่าย** | โดเมน `.com` มัก **หลักร้อยบาท/ปี** (แล้วแต่ registrar) |
| **กี่โดเมน** | แบบง่าย: โดเมนเดียว แยก path (`/admin`) หรือแยก subdomain (`admin.`, `app.`) — บอกว่า CORS/ตั้งค่า backend ต้องตรงกัน |

---

## 5. ถ้าลูกค้าบอก “แพงไป” — สิ่งที่เสนอได้

### 5.1 ลดแพ็กเกจเป็น A

- ย้ายจาก B → A: **รวม Mongo บน VPS เดียว** แทน Atlas  
- **ข้อควรพูดตรงๆ:** “ถูกลง แต่ถ้าเครื่องพังหรือข้อมูลเสีย ต้องพึ่งสำรองที่ตกลงไว้ — แนะนำ backup อย่างน้อยสัปดาห์ละครั้ง”

### 5.2 ลดสเปก VPS

- ลด RAM/CPU ชั่วคราว — **บอกว่า** ถ้าผู้ใช้พร้อมกันเยอะหรือดึงภาพหลายเครื่อง อาจช้าหรือต้องอัปเกรดทีหลัง

### 5.3 แยกค่า “ครั้งเดียว” กับ “รายเดือน”

- **ครั้งเดียว:** ติดตั้ง, ตั้งค่า, อบรมสั้นๆ  
- **รายเดือน:** โฮสต์, โดเมน (ต่อปี), DB (ถ้ามี), ดูแล (ถ้ามีสัญญา)

ลูกค้าจะเห็นชัดว่า “แพง” เพราะส่วนไหน

### 5.4 ไม่ควรลดจนเกินไปโดยไม่บอกความเสี่ยง

หลีกเลี่ยง: โฮสต์ฟรีไม่มี backup, ปิด HTTPS, ใช้รหัส admin อ่อน — **สรุปความเสี่ยงเป็นข้อๆ ให้ลูกค้าเซ็นรับหรือยืนยองในเมล**

## 6. ตารางเปรียบเทียบสั้น (ยกไปวางสไลด์)

| หัวข้อ | ถูก (A) | กลาง (B) | สูง (C) |
|--------|---------|----------|---------|
| ความเสี่ยงข้อมูล | สูงขึ้น | ปานกลาง | ต่ำลง |
| งานดูแลเซิร์ฟเวอร์ | พึ่งทีมคุณ/ลูกค้ามาก | ปานกลาง | แยกชั้น ดูแลซับซ้อนขึ้น |
| เหมาะกับ | ร้านเล็ก / POC | ร้านจริงส่วนใหญ่ | หลายสาขา / เน้นภาพลักษณ์ |

## 7. Checklist หลังประชุม — ให้ลูกค้าเตรียม / ตัดสินใจ

- [ ] เลือกแพ็กเกจ **A / B / C** (หรือปรับกลางๆ)
- [ ] **โดเมน** ชื่อที่ต้องการ (หรือให้คุณช่วยหาให้ที่ว่าง)
- [ ] **บัญชีชำระเงิน** VPS / Atlas (ลูกค้าชำระเอง หรือผ่านคุณ — ตกลงให้ชัด)
- [ ] **อีเมลติดต่อ** สำหรับ SSL / แจ้งเตือน
- [ ] วันเริ่ม **ติดตั้งจริง** และผู้รับผิดชอบฝั่งร้าน (ใครเปิดคอม/สาย USB)
